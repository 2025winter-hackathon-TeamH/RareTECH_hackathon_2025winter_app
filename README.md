# RareTECH_hackathon_2025winter_app_main

◆アプリ名：背水の陣(仮)

## 開発ルール ##
 - main には直接 push しない
 - 作業は各自ブランチで行う
   ブランチ名は以下になるべく寄せる
    - 新機能開発ブランチ：feature_xxx（_xxxは個人名）
    - 開発中ブランチ　　：develop_xxx
    - すぐマージする修正：hotfix_xxx
　　補足：ブランチを push → マージ完了後も、同じブランチ名を再利用してOK
 - 動作確認後に Pull Request（可能な範囲で）

## コミットメッセージルール ##
 - commitメッセージは以下になるべく寄せる
　　- feat: 新しい機能を追加した        　例）feat: loginボタンを追加
　　- fix: バグを直した                　例）fix: ログイン時のエラーを修正
　　- docs: READMEやコメントなど文章変更  例）docs: READMEに起動手順を追記
　　- chore: その他（迷ったらこれ）     　 例）chore: フォルダ構成を整理
